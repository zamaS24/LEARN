>$ git revert <id_commit> : Remmetre le depot à l'etat dans ce commit. 
    Reverting a commit using git revert doesn't remove the original commit from the history. Instead, it creates a new commit that effectively undoes the changes introduced by the specified commit.
    When you revert a commit, Git doesn't delete or remove the commit itself. Instead, it creates a new commit that negates the changes introduced by the original commit. This new commit reflects the inverse of the changes made in the original commit, effectively canceling them out in the current state of the codebase.
    The original commit remains in the history, and the revert commit serves to revert the changes brought in by the original commit while keeping a clear record of this action in the commit history.

Tutorials and extra 

fast-forward merge: 
    When you merge one branch into another in Git, sometimes Git performs what is called a "fast-forward" merge. This occurs when the branch you're merging into (let's say main) hasn't had any new commits since the branch you're merging in (let's say dev) was created.

>$ git push <remote-name> <local-branch-name>:<remote-branch-name>
-> example:
    >$ git push origin test_branch
    This command tells Git to push your local test_branch to the remote repository's test_branch. If the test_branch doesn't exist on the remote repository, Git will create it.
    If you want to push your local branch to a differently named branch on the remote repository, you can specify it explicitly:
    >$ git push origin local_branch_name:remote_branch_name

Merge using GitHub cli 
- add the original repo as an upstream remote 
	>$ git remote add upstream https://github.com/original-owner/original-repo.git
- create the pull request 
	>$ gh pr create --base main --head your-branch-name --repo original-owner/original-repo --title "Your PR Title" --body "Description of the changes"
		--base main: Specifies the branch you want to merge into on the original repository (usually main or master).
		--head your-branch-name: Specifies your branch on your fork (the one with your changes).
		--repo original-owner/original-repo: Specifies the original repository where you want to create the pull request.
		--title: The title of your pull request.
		--body: A description of the changes in the pull request


- review and submit 
	>$ gh pr view --web
